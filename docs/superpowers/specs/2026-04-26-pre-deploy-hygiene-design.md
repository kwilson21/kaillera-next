# Pre-deploy hygiene, doc refresh & commit split

**Date:** 2026-04-26
**Status:** Approved (pending spec review)
**Scope:** Hygiene-only (no structural refactor)

## Goal

Take the working tree from its current state (14 modified files + ~30 untracked
items + stale docs) to a clean, deployable `main` with semantically split
commits and freshened documentation. **No structural refactor of monolith
files.** The deploy itself (`just deploy`) is out of scope — this prep pass
ends at a clean tree on `main` that the user can then deploy.

## Non-goals

- Refactoring `netplay-lockstep.js` (9,518 lines) or `play.js` (4,590 lines).
  Decomposition is tracked separately in `project_monolith_decomposition.md`
  and deserves its own session, not a pre-deploy bundle.
- Running `just deploy` or any deploy step. The user does the Playwright
  two-tab smoke and triggers deploy themselves.
- Editing `CHANGELOG.md`. It is regenerated by `scripts/bump-version.sh` when
  `just deploy` runs (per `reference_deploy_workflow.md`).
- Wiring SoftFloat conversion files or FPU rounding patch into the prod build.
  These are speculative paths not currently compiled into `libsoftfloat.a`;
  wiring them would require a fresh WASM build and full validation cycle.
- Adding new tests. Per `feedback_test_what_fixes_desyncs`, hygiene/diagnostic
  improvements do not require a new test run.

## Sequencing

The order is chosen so each commit is small enough to verify on its own, and
so later commits don't have to step around earlier untracked files.

**Assumed starting state:** working tree matches the gitStatus snapshot
captured at spec-write time — 14 modified files, ~30 untracked items as
listed below. Implementer should `git status --short` and compare before
beginning; if the tree has drifted (new modifications, files moved), pause
and reconcile rather than proceed on the spec's stale assumption.

## Verification gates

After each commit, before moving to the next:

1. `git status` shows only files expected to still be unstaged.
2. `git diff HEAD~1 --stat` matches the commit's intended scope.

**Commit 2 specifically (the only one touching runtime code):** No new long
stress run is required in this hygiene pass; cite the already-run V8/JSC
5-minute stress pass, and run cheap syntax checks if staging changes make
that practical. The runtime fix was validated before this commit-split pass
began.

**Commit 3 specifically:** docs commit must not leak source edits. Run:

```bash
git diff HEAD~1 -- '*.js' '*.py' \
  | grep -E '^[+-][^+-]' \
  | grep -vE '^[+-][[:space:]]*(//|#|/\*|\*|\*/)'
```

This filters for `+`/`-` lines that are NOT comment-block content. Output
should be empty. If anything appears, a non-comment source edit slipped into
the docs commit and must be split out before continuing. Markdown files
(`*.md`) are not subject to this check.

**Final gate, in order:**

1. `git status --porcelain` returns empty (no untracked, no unstaged, no
   staged-but-uncommitted).
2. `git log origin/main..HEAD --oneline` shows the 5 commits in the expected
   order — the user reads this list and confirms it matches the sequence
   above before running `just deploy-dry`.

## Risks & mitigations

| Risk | Mitigation |
|---|---|
| Doc-refresh accidentally edits source code | Commit 3 verification gate; restrict edits to comment blocks + `.md` files |
| WASM trio on `main` without matching patch sources | Already mitigated — `audio-backend-skip-output.patch` is the only prod build input added in commit 2; SoftFloat scaffolding upstream is unchanged |
| Hidden untracked file gets dropped | Pre-step `rm` batch is explicit and reviewable; no glob-based deletions |
| `git status` not actually clean at end | Final gate: `git status --porcelain` returns empty before handoff |
| Doc rewrites inflate scope | Content pass uses "verify and minimally update" wording; if a doc is too far gone, prepend historical note rather than rewrite |
| Module header rewrite invents APIs | Only freshen existing comments; verify referenced files/symbols exist (`grep`) before citing them |

## Handoff

After commit 5, the user runs:

```
just deploy-dry   # preview unpushed commits + version bump
# Playwright two-tab smoke (per feedback_playwright_before_deploy)
just deploy       # bump + push + tag
```

The deploy script generates one `chore(version)` commit with bullets for
the substantive commits in this pass.
